Interface Audit - Landing Page

Visibility of System Status

The page that I will be auditing is the landing page. The page communicates what the page is, which is stated by the title “My interactive sketchbook.” However, I feel as if it doesn’t really give the user much information as to why they are on that page, besides the line of links signaling to the user that it’s a page that leads them to other pages. In that way, I feel like the landing page is unclear.

Affordances & Signifiers

On the landing page, the user has the ability to click on any of the links to view the different sketches, entries, and the development of my micro-project. This action is signified by the links being underlined and in blue. It is also signified by the subheading “Sketchbook Entries. Please feel free to explore.” On the other hand, I can see how it may be unclear to the user what to do, as the action is not specifically stated. There are no terms or phrases like “Click to View” that clearly hint towards the action.

Navigation & Mapping

In regard to navigation and mapping, the landing page feels more like exploration rather than guided. There is no starting point for the user and no end due to there being no real visual cues or phrases like “Start here.” There is also no numbering system, so it could be easy for the user to lose track of which sketch they have visited. The system also doesn’t have a strong structure to it. It feels more like a collection of trial and errors rather than a system with categories. The sketches have no clear grouping, such as projects, exercises, and reflections. Without the visual structure, it makes it hard for the user to know what each link will lead to. The navigation simply does not reflect a clear hierarchy, which leads the user to guess. This can be due to the headings and labels being close in font and size, and having all the links in one line.

Constraints & Assumptions

The page assumes that the user already understands that the landing page is an interactive sketchbook that they know they are supposed to interact with and click on. There are no clear affordances like buttons or arrows, and the user can’t comprehend how the content is grouped. What is hidden within the landing page is that there is no visible organization and no navigation options such as “home,” “menu,” and “back.” In my opinion, the landing page is more of a trial-and-error user interaction. The user doesn’t have a clear understanding of what the links will lead to, and it can cause uncertainty for the user. In regard to restriction, the user is restricted to only being able to click on the links, and that is it.

Heuristic Violations

Three Structural Changes

Three structural changes that I would make to this page are to improve hierarchy, navigation, and interaction clarity. For hierarchy, I would also organize the links by categories such as “Experimental Sketches,” “Terms and Conditions Project,” and “Reflections.” Instead of having them all in one horizontal row, I would have them in a vertical column with spacing between each category to show the difference in hierarchy. This would also help the navigation of the page and improve the interaction clarity. I would add a home button so that way user are able to go back to the landing page. I would also make each link a button as well, and when the user hovers above it, the link would have a visual change, such as color or a movement that symbolizes a pressed-down motion. Besides this, I would also add a description of what the sketchbook is and some phrases like “Click to explore the sketches,” to make it clear to the user what they must do.
