Micro Project - Progress 2

Current Build

In regards to the state of my project, what currently works is the canvas, the scrolling behavior within the text box, and the accept button functionality. However, the scrolling behavior is not a persise as I want it to be. My main variable is "let stage = 0;" which represents the user's current stage in the process of reading and accepting terms and conditions.

Where is the pressure?

The tension in the system is created by the user's interaction with the scrolling and acceptance behavior. The system is treshold-based, since it is designed to change from stage 0 to stage 1 if the user has scrooled passed 250 pixels. Currently it is not accumulative since the reading behavior is does not accumulate over time. However, the system does have restricted access since the user can't accept the terms and conditions without passing a certain amount of pixels. The accept button is greyed (stage 0) and then turns green (stage 1) when it becomes clickable.

One structural addition

This week I will be focusing on keeping the text within the box and making the scrolling behavior more precise. I will also be adding a timer to track dwell time, which will add another layer of tension to the system. The user will have to not only scroll but also spend a certain amount of time on the text to unlock the accept button.
