Interface System - Self-Assessment

Terms and Conditions

Navigation Model and System Coherence - 20

I gave myself a score of 20 due to the consistency of my navigation model between both the laptop and mobile interface, and to show the active and visible states. For example, both interfaces are consistent in the way the user is able to interact with the design. Both use the same arrow icons to navigate the Sketch 9 - Images of Blueberry and are also set up the same way that if the user clicks on the word “HOME” on the mobile interface, they will be taken back to the home page, and if the user is on the laptop interface and clicks home, they can return. However, the key difference is that the user can’t navigate to the Terms and Conditions page or the Sketch 9 - Images of Blueberry page through the words on the tab list at the top of the mobile interface, whereas on the desktop interface, they can. I made this decision because I thought it would make the mobile interface feel less cluttered.

Hierarchy & Mobile-First Logic - 17

I gave myself a score of 18 because I believe that my design interface does show levels of hierarchy, but I feel as though the Headings like “HOME” do not show much differentiation, like the subheadings such as “How to explore,” “Terms and Conditions,” and “Sketch 9.” This is due to both the heading(s) and the sub-headings having the same font size. However, besides that, there is differentiation between the title, sub-headings, and the body text due to the sizing, if the words are boxed in with other information, are bolded or regular, or are positioned to be on top.

Explain how hierarchy survives compression and expands across contexts - 13

I gave myself 13 points because my work clearly shows me attempting to make my mobile interface survive compression by making the Terms and Conditions page longer, and I also attempted to make my laptop expand across contexts. However, I don’t think I achieved this successfully, as the relationship between the compressed and expanded states is not always obvious to the user; it sometimes feels like separate layouts rather than the same system evolving. I think the key problem is that the users wouldn’t easily recognize the same hierarchy across contexts. But I will say that the hierarchy is consistent across the mobile and laptop due to the small supporting icons such as dots, captions, and overlays. I also have the correct compression logic when it comes to the images (boxes of information) because for the mobile, it is a single image focus, whereas for the desktop, there are multiple images in a row.

Theoretical Integration (Norman / Nielsen / Shneiderman) - 13

I gave myself 13, because I think I did very well at implementing Nielsens heuristics. I implemnted consistency with repeated layouts and small gallery structure, I implemented recognition over recall by making it easy for the user to understand how to navigate through the system and making it familiar to the user from previous interactions, I also implemented user control and freedom by having multiple paths that can lead to a page or more than one page, and I also added in flexibility and efficiency (partially because the users can’t navigate to other pages through the tab section on the mobile interface) by providing the user different ways to navigate. Regarding Shneiderman’s Golden Rules, I have been able to achieve consistency, shortcuts, support of internal locus of control, and reduce short-term memory load. Norman principles are also present by having affordances with some UI suggesting interaction; however, this is never really clear. For example, the words in the tab don’t hint to the users that they are clickable. Also, my feedback is only present when the users hit accept on the Terms and Conditions page.

Cross Page Consistency - 10

I believe my sketchbook does document code experiments, rule changes, and system adjustments. Each of my sketchbooks contains rules learned in class and development in complexity. For example, “number 1” uses mouse x and mouse y keys, and the draw function and in a later sketchbook, “number 9” use and auto-advanced mode to switch between photos, which didn’t involve user in put.

Iterative Development - 10

Compared to my early drafts, my work has indeed improved. For example, I made the interface much easier to navigate by adding shortcuts such as the tab menus at the top, and also made sure that the user got feedback on the status of the page, and simplified how the user could navigate the page by implementing the forward and back arrows. In the beginning, my design lacked mobility to move between pages, but now it is visible to the user and easy to use. Unfortunately, I have been working on the same draft, so I am not able to present the earlier versions, but I really enjoyed the time spent getting feedback and revising the interface to be more user-friendly. I do have photos that I can upload.

Participation and Critique Engagement - 5

I was an active participant during the critiques, and I was also able to implement the feedback that I was given. One of the main pieces of feedback was to not include the other sketches on the Home tab because it can confuse the user, so I got rid of them. Another piece of feedback I got was actually how to connect the screens/pages so that it works in the prototype. Overall, the feedback and critique were a great experience.
