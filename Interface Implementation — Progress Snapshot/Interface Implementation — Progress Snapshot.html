Interface Implementation - Progress Snapshot

Reflection

Regarding what I have implemented so far, I wanted to focus on getting the basic organization down before I started to fully design. This included making sure that each of the pages that I worked on in Figma had the title “Mel’s Interactive Sketchbook,” and the linked “tabs” at the top, which are just links for right now, “Home, Terms and Conditions, Sketch 9: Images of Blueberry.” The majority of my time was spent on the home page, where I turned every other sketch and reflection into buttons, which I spaced out to make things more organized. Structurally, it is all pretty much working; the links all work, and there is a clear understanding of the flow of information. With my Figma design, I focused on shortcuts (the tabs), which I didn’t have before, so it logically makes sense to the user that if they wish to navigate through the system, they can click on the tabs. The only breaking points I ran into were that my CSS was not working, but that was because of a linking issue in my HTML.

While coding, what surprised me was how much of what we previously learned helped in re-designing our code. For example, I used what I learned from my Terms and Conditions project regarding making the square and centering it for the bottom buttons of the Home page. I just tweaked it a bit so that it worked for multiple squares instead of just one. I think overall it was harder to design with CSS only because I am not too familiar with it, but with the template provided and some research, I was able to get the hang of it. To be honest, I still kinda don’t understand some of it, but the code works! I did notice that while designing, the browser did show me that another shortcut is just clicking the back button on the window menu, but that it is more efficient for the user to click on the tabs I provided for a quicker shortcut. My interface assumes that my layout is easy for the user to understand and navigate through the organization of it. Although right now it is a bit unclear, the user can tell that there is a main focus on the Terms and Conditions project and the Sketch 9 - Images of Blueberry. I think CSS assumes that my design has a document-style kind of flow to it, unless told by implementing a flexbox or positioning, especially when it comes to the navbar. On the other hand, HTML assumes a clear structure due to the heading, body, and paragraphs; because of this, it relies on hierarchy in order for it to be organized. When coding, it was hard to decide which design choices belonged to HTML and which belonged to CSS. But it was easy to understand that HTML did the basic organization of things like headings and subheadings, and CSS designed it visually so that it is not an organized mess. For example, my design included elements that needed a container, to not cause layout issues, because the browser didn’t “know” where it should belong visually.

In order for my work to be ready for the final submission, I need to work on centering the title and the subheading with a distinct design and formatting it so that it resembles my Figma design. I also need to focus on implementing the mobile design as well. There are also the pop-up windows and refresh buttons that I need to implement for my terms and conditions project.
