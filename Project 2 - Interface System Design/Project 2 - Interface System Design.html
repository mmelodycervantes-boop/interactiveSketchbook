Project 2 - Interface System Design

Statement

When it came to designing the mobile and computer interfaces, I wanted a clear navigation system. For the mobile and the computer, I wrote a “How To Explore” section so that the user has a better idea of what to expect from the interactive sketchbook. I made navigation easy by using the words and buttons as a form of navigation. When the user clicks on a section title such as “HOME,” “Terms and Conditions,” and “Sketch 9,” they are then moved to the next page. For example, if the user wanted to go back to the home page, they would simply click on “HOME.” If the user wants to switch between pictures of Blueberry, they can use the back and forward arrows. For the Sketch 9 - Photos of Blueberry, I also added dots in white and grey, so that the user knows the status of where they are in the array of photos. When it came to hierarchy, I used four different hierarchies for the mobile and three for the laptop. For the mobile, there is the title, headers, body text, and the indication text (forward and back). For the laptop, there is the title, headers, and body text. I also used uppercase and lowercase so that the user knows what the hierarchy is when it comes to the subheaders, such as “HOME” and “Sketch 9.” To be honest, I am glad we switched over to doing the mobile prototype first, as I found it to be more complex due to the number of screens needed. Since I used my phone more often than my laptop, I could use my own experience to design the mobile interface. After figuring out how the screens connected through the mobile interface, I could easily translate it into a laptop interface. Designing the mobile interface was also much easier, as the design didn’t take up too much space, so I didn’t have to worry about the negative space. However, with the laptop interface, I found it harder, as I had a lot more space to take up. In reference to Norman, my design does a good job of addressing discoverability and signifiers. Norman emphasized that users should immediately understand what actions are possible without instructions. In my design, I added elements that can be clicked on, such as the triangle button, accept buttons, navigation icons (the i-icon, arrows, and x-box). I also made the navigation tab, which can be differentiated through color contrast (in grey scale), and is placed in the tab section. There are also dots that tell the user which photo in the array they are looking at. In reference to Nielsen, my design aligns with Nielsen’s 10 heuristics, especially visibility of system status, consistency and standards, and recognition rather than recall. In my design, I have the progress dots and confirmation of the acceptance of the terms and conditions, so that the user always knows what is happening. I also have standard user interface patterns such as icons (a triangle for drop-down and the arrows for back and forward), so that the user doesn’t have to relearn interactions. My design also does well at redirecting the user's need to memorize everything, as the design itself is easy to navigate due to the visibility of actions. In reference to Schneiderman, my design reflects Schneiderman's principles due to consistency, informative feedback, and shortcuts. In my design, I have consistent colors even though they are in greyscale to show what the headers and tabs are, and to help the user identify what is clickable. There is also similar typography in upper and lower case across both mobile and laptop interfaces. My design also clearly responds to the user's actions. On the laptop, I also added shortcuts for the user to navigate fast by utilizing the tab section at the top. In designing both the mobile and laptop interface, I designed it with the intention that it would be easy for the user to understand how to navigate between the pages. To do this, I made sure that each page looks almost identical and functions the same. This helps to reduce cognitive load early on in the user’s experience with the interface. Each button has a page that it leads to, and if the user wishes to return to their previous page, it is easy for the user to understand the buttons that can make that action possible, such as the arrow icons, the “HOME” button, the refresh button, and the exit button (x-box).
