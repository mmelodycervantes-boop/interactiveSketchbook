Reading Reflection 3

Which rule feels most absent from your micro-project page?

My micro-project lacks consistency. In analyzing my micro-project, I realized that it feels more chaotic than consistent with layouts and visual patterns, which forces the user to relearn interactions on each screen. There is no clear how-to, but rather it feels chaotic and overwhelming, which is how it was intended to be. I wanted the user to feel frustrated/overwhelmed, but perhaps I have confused them instead. Some of the elements in my micro project don’t always behave predictably; in some way, the buttons and interactions don’t feel like part of one whole planned-out system. There is a lot of uncertainty.

Which rule would most improve your sketchbook interface?

I think reducing memory load would improve my sketchbook interface. Currently, my sketchbook is designed to have the user click on the links, not knowing where those links lead. There are no clear categories as to what the links are, which makes the user have to rely heavily on their memory to know what each link is. I think that putting categories or spacing out the links, it can help the user experience.

Which rule is hardest to implement, honestly?

I think the rule that might be the hardest for me to implement would be to permit easy reversal of actions. I think this rule will be hard to enforce because the whole point of my micro-project is for the user to gain access to click the button. If the user can reverse their actions, then it kinda ruins the point of the micro-project. The micro-project is meant to correct the users' actions so that they become trained to read the terms and conditions.

Did any rule contradict how you designed your system originally?

I think the main rule that contradicts my design is error prevention and user control. Both contradict my original design as these rules aim to reduce frustration, prevent mistakes, and support smooth interaction. However, the system that I created is intentional in the fact that it causes the user to experience a sense of friction in order to make the user guilty for skipping the terms and agreement. Instead of preventing errors, the design is meant to fix the users' errors in order for them to learn to read the terms and conditions, not just in my design, but in other places as well.
